## Where we are (0.2.0 baseline)
Litmus ships the full **Tier-1** manufacturing-test set: First Pass Yield, final
yield, failure pareto (by step/measurement), test-time stats, Cpk/Cp, yield trend,
retest rate, time-loss. That's a solid, complete baseline ‚ÄĒ the gaps are Tier-2
quality-engineering metrics.

## Additions (prioritized)

### Quick wins ‚ÄĒ derived computations on existing tabs
- **Ppk / Pp** (Cpk tab) ‚ÄĒ long-term *performance* (overall ŌÉ) alongside Cpk's
  short-term *potential* (within-subgroup ŌÉ). Every quality sign-off (PPAP/APQP,
  automotive/medical) requires both; the Cpk‚ÜĒPpk gap is a stability signal.
  `Ppk = min((USL‚ąíxŐĄ), (xŐĄ‚ąíLSL)) / 3ŌÉ_overall`.
  *Design Q: does today's Cpk use within-subgroup ŌÉ or overall ŌÉ? If overall, Cpk‚ČąPpk
  today ‚ÄĒ resolve before adding so they're genuinely distinct.*
- **RTY (Rolled Throughput Yield)** (yield tab) ‚ÄĒ `‚ąŹ FPY_step`; reveals compound
  loss FPY hides (5 stations √ó 95% = 77%). *Design Q: define the "sequence" for
  Litmus's data model ‚ÄĒ per-step within a run's step sequence vs per-station across a
  multi-station flow (Litmus may not model multi-station flows yet).*
- **DPMO / DPPM** (yield / pareto) ‚ÄĒ `defects / (units √ó opportunities) √ó 1e6`;
  opportunities = test steps. Normalized defect unit for cross-product comparison +
  customer targets (auto <50 DPPM). Well-defined for Litmus's steps data.

### Features ‚ÄĒ new views (own design pass)
- **Per-measurement SPC control charts** (I-MR / X-bar-R + Western Electric / Nelson
  rules) ‚ÄĒ plots a measurement in run order with control limits, flags drift *before*
  it crosses spec. Highest-value addition for test engineers ("catch drift before
  yield drops"); in every serious tool (Sciemetric, WATS). Cpk is a snapshot; SPC is
  the movie.
- **Yield cross-tab by station / fixture / operator / shift** ‚ÄĒ the #1 diagnostic
  drill-down: "is it the *product* or the *fixture*?" A flat FPY can't answer it.
  Needs station/fixture/operator stamped on each run (mostly present).

### Refinements to the existing 6
yield ‚Üí add RTY + DPMO; cpk ‚Üí add Ppk/Pp + show the Cpk‚ÜĒPpk gap; trend ‚Üí add
per-measurement SPC view; failure_pareto ‚Üí absolute counts (for DPMO) + a
by-dimension (station/operator/date) sub-view; retest ‚Üí per-station/fixture (not just
per-serial); time_loss ‚Üí station utilization % (a lightweight OEE proxy).

## DOCS REQUIREMENT (hard rule ‚ÄĒ per user, 2026-06-21)
**Every metric added in 0.3.0 must ship with updated docs in the same change:**
- new/refreshed **operator-UI screenshots** for the affected tabs (run
  `scripts/regenerate-ui-screenshots.py`), AND
- a **curated, representative demo dataset** behind them ‚ÄĒ not the current thin seed.

The current screenshot seed (`examples/07-profiles`, re-seeded by `pytest`) is a poor
analytics demo: it is **all-passing** (so failure_pareto renders an empty state) and
its accumulated data dir carries **stale runs with ~141-hour durations** (garbage
test-time stats). 0.3.0 needs a deliberate demo dataset: real **failures** (for
pareto/DPMO), **realistic durations**, **multiple serials/stations/shifts** (for the
cross-tab), and a **time series** (for trend/SPC). Build that dataset as part of the
analytics work so the screenshots actually show the metrics doing their job.
