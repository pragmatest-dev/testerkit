## Findings

### Marginal — `docs/reference/parquet-schema.md:479`

> "`vector_retry` is **0-based** … Filter to the final attempt with
> `WHERE vector_retry = (SELECT MAX(vector_retry) …)` …"

Uses "attempt" in English prose, not as an API name. The new vocabulary
talks about "executions" (first execution + N retries). Replacing
"final attempt" with "final execution" tracks the API names without
breaking sentence structure.

**Status:** closed by the alignment commit (this audit's commit).

### No-op — `docs/guides/writing-tests.md:195–222`

Section already updated in commit `1268951` to use the new flat shape:

```yaml
connections: [vout_measure]                     # by name
connections: {dmm: ["ch1", "ch2"]}              # by channel
```

A scoping pass flagged the prose framing on line 202 as potentially
wrapper-relative, but on re-read the framing stands alone — Pydantic
discrimination is explicit on line 197 and both bullet points use the
post-rename shape. No change needed.

## Out of scope (intentionally deferred)

- **`docs/explorations/*` references** to old table names. These are
  design / parking docs, not user-facing. Updating them belongs in a
  broader exploration-doc cleanup, not the 0.1.0 alignment.
- **Cross-reference enrichment.** The new `concepts/step-hierarchy.md`
  is internally consistent with the rest of the concepts tree and
  has working cross-links from `guides/writing-tests.md` and
  `guides/vector-expansion.md`. Adding more cross-links is editorial
  taste, not alignment work.
- **Tutorial code-flow drift.** Identifier renames are clean; whether
  the tutorial's broader narrative still tracks current behavior is a
  separate audit (effort: half-day to walk the 10 tutorial pages).
- **CLI completeness check.** Whether `reference/cli.md` covers every
  current `litmus` subcommand and flag is its own audit, not part of
  the rename-hotspot pass this document records.

## Conclusion

Docs ship cleanly with the 0.1.0 cut. One single-word prose tweak
closed inline.
